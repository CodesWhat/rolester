## Initial setup

Configuration is produced by `ingest-profile` during onboarding. You do not
hand-edit the config files directly during setup — the agent interviews you and
writes the files via validated CLI helpers.

## Changing settings later

To change any setting after onboarding without re-running the full interview:

> change a setting / update my comp floor / add an excluded company
> turn on browser automation / change my writing style

The agent routes to `configure`, which:

1. Shows your current state (`npm run doctor`, `npm run modes -- status`,
   `npm run automation -- status`).
2. Routes each change to the canonical validated CLI helper (`npm run gate`,
   `npm run modes`, `npm run automation`) or the owning skill.
3. Confirms before writing consequential changes.

`configure` never becomes a separate mutation path — it routes through the same
validated tools that the skills use.

## Gate files and the gate helper

When you state a new constraint mid-flow ("never that company", "below $190K is a
no"), the agent writes it to the canonical config file so all other skills inherit
it immediately. It uses `npm run gate`:

```bash
npm run gate -- exclude-company "Acme Corp"           # dry-run
npm run gate -- exclude-company "Acme Corp" --write   # commit
npm run gate -- comp-floor 180000 --write
npm run gate -- cut-signal "requires >25% travel" --write
```

Gate types: `exclude-company`, `cut-signal`, `keep-signal`, `comp-floor`,
`comp-target`, `comp-expected`, `do-not-claim`, `do-not-fabricate`.

## Mode switches

`candidate/modes.yml` (optional) controls operational posture. If absent,
defaults are `standard` usage, `balanced` application mode, and `standard` voice.

```bash
npm run modes -- status                        # current values
npm run modes -- set usage_mode lean --write   # change usage mode
npm run modes -- set application_mode high-volume --write
npm run modes -- set agent_voice exec-summary --write
```

| Mode | Options | Effect |
|------|---------|--------|
| `usage_mode` | `lean`, `standard`, `full` | How much discretionary work Rolester runs (research, deep packets, board discovery) |
| `application_mode` | `selective`, `balanced`, `high-volume` | Pursuit posture after discovery (thresholds, medium-fit review) |
| `agent_voice` | `exec-summary`, `standard`, `technical`, `verbose` | How the agent talks to you |

No mode relaxes the honesty, privacy, comp, or application-limit gates.

## Browser automation switches

```bash
npm run automation -- status                         # the live matrix
npm run automation -- consent linkedin --write       # record ToS consent
npm run automation -- enable status_polling all --write
npm run automation -- session extension --write      # preferred provider
```

See [Browser Automation](/advanced/browser-automation) for the full capability
list and consent model.

## Config files reference

| File | Managed by | Purpose |
|------|------------|---------|
| `profile.yml` | `ingest-profile`, `configure` | Identity, comp, location, domain |
| `targeting.yml` | `ingest-profile`, `npm run gate` | Role targets, keep/cut signals, exclusions |
| `evidence.yml` | `ingest-profile`, `npm run evidence` | Accomplishment claims |
| `honesty.yml` | `ingest-profile`, `npm run gate` | Honesty boundaries |
| `form-defaults.yml` | `ingest-profile`, `configure` | Portal form answers |
| `application-limits.yml` | `configure`, `npm run gate` | Per-company caps and cooldowns |
| `modes.yml` | `npm run modes` | Usage and voice posture |
| `automation.yml` | `npm run automation` | Browser capabilities |
| `search-sources.yml` | `setup-searches`, `research-boards` | Job board and ATS sources |
| `sourced-scan.json` | `discover-companies` | Tracked companies for the sweep |
