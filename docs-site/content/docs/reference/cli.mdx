## rolester CLI (bin/rolester.mjs)

The primary entry point. Run as `node bin/rolester.mjs <command>` from the repo
root, or via the `rolester` binary if you have it on PATH.

| Command | What it does |
|---------|-------------|
| `start [agent]` | Scaffold workspace, install skills, boot dashboard, launch agent |
| `start claude` | Same, with Claude Code as the agent |
| `start codex` | Same, with Codex as the agent |
| `init` | Scaffold workspace only (no agent, no dashboard) |
| `doctor` | Environment health check, scaffold state, skills discoverability |
| `ingest` | Alias for running `ingest-profile` via the agent |
| `evaluate` | Alias for running `evaluate-job` via the agent |
| `tracker` | One-shot render of dashboard to `workspace/tracker.html` |
| `update` | Pull latest code from npm; data untouched |
| `restore` | Recover `tracker.json` from a rolling snapshot (confirm-first) |
| `--version` | Print installed version |

### start flags

| Flag | Effect |
|------|--------|
| `--no-agent` | Scaffold + dashboard only |
| `--no-dashboard` | Agent only, no dashboard server |
| `--agent <name>` | Override agent CLI (any CLI on PATH) |
| `--port <n>` | Dashboard port (default 7777) |

## npm scripts (package.json)

Run from the repo root with `npm run <script>`.

### Core

| Script | What it does |
|--------|-------------|
| `tracker` | Render dashboard snapshot to `workspace/tracker.html` |
| `tracker:dev` | Live-reloading dashboard server on `:7777` |
| `doctor` | Environment health check |
| `ingest` | Run `ingest-profile` flow via agent |
| `update` | Pull latest code from npm |

### Config management

| Script | What it does |
|--------|-------------|
| `gate` | Read or write a candidate gate (comp floor, exclusion, signal) |
| `modes` | Read or set `usage_mode` / `application_mode` / `agent_voice` |
| `automation` | Read or toggle browser automation capabilities and consent |

### Data management

| Script | What it does |
|--------|-------------|
| `evidence` | Add or validate evidence claims |
| `learnings` | Manage per-role-family learning files |
| `stories` | Manage STAR+R story bank |
| `activity` | Append or manage Activity Pulse events |
| `research` | Record or read research artifacts |
| `strategy-review` | Stamp a strategy-review run |

### Validation

| Script | What it does |
|--------|-------------|
| `verify:tracker` | Validate `workspace/tracker.json` schema and integrity |
| `install-skills` | Re-shim `.claude/skills` → `.agents/skills` |
| `install-skills:check` | Report whether skills are discoverable |

### Capture and export

| Script | What it does |
|--------|-------------|
| `capture:search-sources` | Bulk-capture board listing pages |
| `capture:board` | Capture a single board page |
| `export` | Export a markdown artifact to PDF (or `.docx`) |

### Advanced

| Script | What it does |
|--------|-------------|
| `searches` | Build search URLs from `config/search-sources.yml` |
| `analyze-outcomes` | Read outcome analytics from tracker |
| `check-job-links` | Check liveness of sourced job links |
| `backfill-benefits` | Backfill benefits data on tracker rows |
| `restore` | Recover tracker from rolling snapshot |

## Environment variables

| Variable | Default | Purpose |
|----------|---------|---------|
| `ROLESTER_HOME` | `<repo root>` | Override location of `candidate/` and `workspace/` |
| `NEXT_PUBLIC_SITE_URL` | *(unset)* | Docs-site base URL for metadata |
